typedef shyft::time_axis::generic_dt shyft::api::gta_t

generic time-axis

time-series math to be exposed to python

This provide functionality like

a = TsFactory.create_ts(..) b = TsFactory.create_ts(..) c = a + 3*b d = max(c,0.0)

implementation strategy

provide a type apoint_ts, that always appears as a time-series.

It could however represent either a point_ts<time_axis:generic_dt> or an expression like a abin_op_ts( lhs,op,rhs)

Then we provide operators: apoint_ts bin_op a_point_ts and double bin_op a_points_tsUsing the time_axis library and concept directly. This time-axis is generic, but currently only dense-types fixed_dt (fastest) calendar_dt( quite fast, but with calendar semantics) point_dt ( a point at start of each interval, plus the end point of the last interval, could give performance hits in some scenarios)
