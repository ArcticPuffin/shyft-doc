Detailed Description

template<class S, class D, class P, class G, class SC>
struct shyft::core::inverse_distance::temperature_model< S, D, P, G, SC >

Provide a minimal temperature model for the IDW algorithm, providing functionality for transformation of source temperature to destination location using computed temperature gradient, based on nearby sources for that temperature location. TODO: Add more robust handling for source stations, value range etc.

Template Parameters
SSource ref. IDWInterpolation template parameter
DDestination ref. IDWInterpolation template parameter
PConfiguration Parameters, here we just need the default_gradient (C/m):
  1. .default_gradient() const;
GGeoPoint type that supports:
  1. static member G::distance_measure(a,b,p)
  2. .z (todo maybe use .height()const instead

Implemented IDW required members are:

  1. class scale_computer with .add(S,t) and .compute(), including default gradient if to few points
  2. static method .distance_measure(GeoPointType, GeoPointType, double, double)
  3. static method .transform(double sourceValue,double scale,const S&source,const D& destination)
    See also
    IDWTemperatureParameter for how to provide parameters to the IDW using this model
